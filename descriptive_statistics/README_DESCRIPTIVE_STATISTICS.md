# Descriptive Statistics

This directory contains scripts for extracting and analyzing GitHub noreply users from World of Code (WoC) data, computing descriptive statistics, and visualizing commit timestamp data.

All the scripts in this directory have to be run on the World-of-Code servers. Please see the note in Step 1 of `GitHub_dataset_creation/README_GITHUB_DATASET_CREATION.md` on how to get access to these servers.

## Overview

The scripts in this directory work together to:

1. Extract GitHub noreply users from WoC datasets
2. Process and analyze unique projects and commit data
3. Aggregate temporal statistics
4. Visualize commit patterns over time

## Data Extraction and Processing Scripts

### extract_noreply_users_from_woc.py

Comprehensive Python script that extracts and processes GitHub noreply users from WoC data.

**What it does:**

- Searches WoC `a2PFull` files for `@users.noreply.github.com` email addresses
- Extracts and normalizes author information
- Processes username variants (merges `id+username` with plain `username`)
- Generates multiple output files with unique elements and statistics

**Usage:**
```bash
./extract_noreply_users_from_woc.py
```

**Output files:**

- `WoC_githubNoreplyUsers_a2P.csv` - Raw author-to-project data
- `WoC_githubNoreplyUsers_uniqueAuthorStrings.csv` - Unique author strings
- `WoC_githubNoreplyUsers_uniqueEmailStrings.csv` - Unique email addresses (cleaned)
- `WoC_githubNoreplyUsers_uniqueUsernameStrings.csv` - Unique username strings
- `WoC_githubNoreplyUsers_uniqueUsernames.csv` - Merged usernames with variant information
- `WoC_githubNoreplyUsers_uniqueNumericIds.csv` - Unique numeric IDs from GitHub
- `WoC_githubNoreplyUsers_uniqueProjects.csv` - Unique projects with noreply commits
- `WoC_githubNoreplyUsers_summary.csv` - Summary statistics

**Requirements:**
- Access to WoC data at `/da?_data/basemaps/gz/`
- Python 3 with standard libraries

### extract_unique_P.py

Utility script to extract unique values from the second column of a semicolon-separated CSV file.

**Usage:**
```bash
./extract_unique_P.py <input_file> <output_file>
```

**Example:**
```bash
./extract_unique_P.py WoC_githubNoreplyUsers_a2P.csv WoC_githubNoreplyUsers_uniqueP.csv
```

**Purpose:**
Extracts unique project names from the author-to-project mapping generated by `extract_noreply_users_from_woc.py`.

---

### getCommitTimestamps.sh

Retrieves commit timestamps for commits made by noreply users from WoC.

**Usage:**
```bash
./getCommitTimestamps.sh > commit_timestamps.txt
```

**Process:**

1. Searches WoC `aFull` files for noreply authors
2. Maps authors to commits using `a2c`
3. Retrieves commit dates using `c2dat`
4. Outputs timestamps (one per line)

**Requirements:**

- WoC lookup tools in `~/lookup/`
- Access to WoC data version V2412

---

### getUniqueP_WoC.sh

Calculates statistics about unique projects in WoC.

**Usage:**
```bash
./getUniqueP_WoC.sh
```

**Metrics computed:**

1. Total unique deforked projects in WoC
2. Total unique deforked GitHub projects in WoC (excluding paths with `/`)
3. Number of projects with at least one noreply email
4. Number of GitHub projects with at least one noreply email

**Note:** This script expects `WoC_githubNoreplyUsers_uniqueP.csv` to exist (generated by `extract_unique_P.py`).

---

### sum_years.sh

Aggregates commit counts by year from multiple parallel processing outputs.

**Usage:**
```bash
./sum_years.sh <base_filename>
```

**Example:**
```bash
./sum_years.sh timestamps
# Processes: timestamps_0.csv, timestamps_1.csv, ..., timestamps_127.csv
```

**Input format:**

- 128 CSV files with space-separated values: `<count> <year>`
- Files numbered from `_0.csv` to `_127.csv`

**Output:**
- `aggregated_by_year.csv` - Comma-separated file with columns: `count,year`
- Sorted by year in ascending order

**Purpose:**
Combines results from parallel timestamp processing to create a single aggregated dataset.

## Visualization Scripts

### plot_timestamps.R

R script for creating comparative visualizations of commit timestamp data.

**Usage:**
```bash
Rscript plot_timestamps.R <file1.csv> <file2.csv>
```

**Example:**
```bash
Rscript plot_timestamps.R aggregatedbyyear.csv aggregatedbynoreply.csv
```

**Input format:**
Both CSV files should have two columns:

- Column 1: `count` (number of commits)
- Column 2: `year`

**Output files:**

- `WoC_commitTimestamps_comparison_plot.png` - High-resolution PNG (300 DPI)
- `WoC_commitTimestamps_comparison_plot.pdf` - PDF format
- `WoC_commitTimestamps_comparison_plot.tex` - LaTeX/TikZ format

**Features:**

- Line plot comparing two datasets
- Color-coded series (blue and red)
- Summary statistics for both datasets
- Publication-ready output formats

**Requirements:**

- R with packages: `ggplot2`, `dplyr`, `tikzDevice`

**Installation:**
```r
install.packages(c("ggplot2", "dplyr", "tikzDevice"))
```

---

### plot_timestamps.sh

Convenience wrapper script for `plot_timestamps.R`.

**Usage:**
```bash
./plot_timestamps.sh
```

**What it does:**
Calls `plot_timestamps.R` with predefined input files:

- `aggregatedbyyear.csv`
- `aggregatedbynoreply.csv`

**Note:** Ensure the input CSV files exist before running this script. To generate the input files, make sure to run `sum_years.sh` once on all commit data, and once on the users-noreply-commit data.

## Utility: WoC Author Lookup

### woc-a2A-lookup-parallel-xargs.sh

**This script is independent from the noreply user analysis workflow described above.**

Parallelized script for performing author-to-author (a2A) lookups in WoC using `xargs` for parallel processing. This is used for all evaluations that use ALFAA. See the README files in the directories `GitHub_dataset_evaluation` and `tool_evaluation` on how to generate the necessary input files, how to run this script, and what to do with the output files.

**Usage:**
```bash
./woc-a2A-lookup-parallel-xargs.sh <input_file> <output_file> [num_jobs]
```

**Parameters:**

- `input_file` - File containing author strings (one per line)
- `output_file` - Output file for lookup results
- `num_jobs` - Number of parallel jobs (optional, defaults to number of CPU cores)

**Example:**
```bash
./woc-a2A-lookup-parallel-xargs.sh authors.txt author_mappings.txt 16
```

**Features:**

- Parallel processing using all available CPU cores
- Handles special characters in input lines
- Progress reporting
- Sorted output
- No dependency on GNU parallel (uses `xargs` instead)

**Requirements:**

- WoC lookup tools in `~/lookup/`
- Access to WoC data version V2409

**Output:**

- Sorted list of author-to-author mappings from WoC

## Typical Workflow

For analyzing GitHub noreply users in WoC:

1. **Extract noreply users and projects:**
   ```bash
   ./extract_noreply_users_from_woc.py
   ```

2. **Extract unique projects (optional):**
   ```bash
   ./extract_unique_P.py WoC_githubNoreplyUsers_a2P.csv WoC_githubNoreplyUsers_uniqueP.csv
   ```

3. **Get commit timestamps:**
   ```bash
   ./getCommitTimestamps.sh > noreply_commit_timestamps.txt
   ```

4. **Process timestamps in parallel** (split into 128 chunks and process - not shown here)

5. **Aggregate results by year:**
   ```bash
   ./sum_years.sh timestamps
   ```

6. **Generate comparison plots:**
   ```bash
   ./plot_timestamps.sh
   ```
   Or with custom files:
   ```bash
   Rscript plot_timestamps.R aggregatedbyyear.csv aggregatedbynoreply.csv
   ```

7. **Get project statistics:**
   ```bash
   ./getUniqueP_WoC.sh
   ```

## Notes

- Scripts assume WoC data version **V2412** unless otherwise specified
- The lookup script (`woc-a2A-lookup-parallel-xargs.sh`) uses WoC version **V2409**
- Large datasets may require significant processing time and storage
- Parallel processing scripts will use all available CPU cores by default
- Output files use consistent naming with `WoC_githubNoreplyUsers_` prefix
